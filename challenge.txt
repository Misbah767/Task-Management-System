JWT (JSON Web Token) ek secure string hota hai jo authentication ke liye use kiya jata hai.
Ye teen parts me divide hota hai â€” Header, Payload, aur Signature.

Header me token ka algorithm (jaise HS256) aur type store hota hai.

Payload me user-related data hota hai, jaise userId, iat (issued at), aur exp (expiry).

Signature token ka sabse important part hota hai. Ye ensure karta hai ke token tamper (modify) nahi hua.

Jab server token generate karta hai, wo header aur payload ko combine karke apni secret key ke sath sign karta hai.
Ye signature mathematically proof hoti hai ke token server ne hi banaya hai.

Jab client dubara request bhejta hai token ke sath, server token verify karta hai.
Wo dobara header + payload ko use karke same secret se signature generate karta hai aur dono compare karta hai.
Agar dono signatures match karein aur expiry valid ho â€” token authentic aur valid hai.
Agar signature mismatch kare ya token expire ho gaya ho â€” to request reject ho jati hai.

Is tarah JWT ka signature part hi guarantee karta hai ke koi bhi token me chher-chaad nahi kar sakta.
Ye poora system stateless hota hai, matlab server ko session store karne ki zarurat nahi padti.
